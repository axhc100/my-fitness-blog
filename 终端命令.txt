本地测试和一键上线两套指令，你直接在 VS Code 的终端里复制运行即可：
💻 1. 本地预览指令
npm run dev

🚀 2. 上传云端指令（Git 提交 + 自动部署）
本地预览觉得完美无误后，按 Ctrl + C 退出预览，然后依次轰入以下三行标准 Git 命令。只要你一推上去，线上的 Vercel/Netlify 就会自动捕捉并帮你更新网站，完全不需要手动去网站后台操作！
# 1. 把刚刚新建的两篇 .md 文章文件添加到暂存区
git add .

# 2. 提交文件，并附带一条更新日志（告诉自己今天加了新文章）
git commit -m "content: add summer fat loss article for zh and en"

# 3. 开启推进！把本地代码和文章一并推送到云端仓库
git push origin main


📝 Next.js 博客高效发文流三步法
1. 生产 Markdown
在 content/zh/ 和 content/en/ 下创建相对应的博文文件。

⚠️ 硬核细节：文件名必须全小写、不留空格，多单词之间使用连字符 - 连接（例如：fasted-cardio.md）。

2. 🚀 第一步：利用 GitHub 网页端上传图片（秒拿 CDN 链接）
当你写文章需要插入高清健身拉伸图、对比图时，直接使用你建好的 my-blog-images 仓库：

拖拽上传： 浏览器打开你的 my-blog-images 仓库，点击 Upload files，把图片直接拖进去，点击 Commit changes 提交。

直链公式： 记住你的专属 jsDelivr 加速公式（把最后的 图片名.jpg 换成你刚传的图即可）：

Plaintext
https://cdn.jsdelivr.net/gh/axhc100/my-blog-images@main/你的图片名.jpg

📝 第二步：本地或任意地方写 Markdown（规范 Front Matter 头）
在你的本地项目 content/zh/ 或 content/en/ 下面，你可以直接建单文件，也可以建子文件夹（如 2026-06/）来归类管理。

新建一个 .md 文件（例如 abs-training-guide.md），文章顶部必须严格包含如下的 Front Matter 格式（这是给代码解析标题和日期用的）：

Markdown
---
title: "2026硬核腹肌撕裂者指南：如何降低体脂露出腹肌"
date: "2026-06-19"
description: "结合三大宏量营养素计算器，通过科学有氧与抗阻训练，在两周内打造清晰腹肌线条。"
---
